Detailed Descriptions of MILES Options

contol (real) convergence tolerance

Whenever an iterate is encountered for which epsilon(z) < CONTOL, the algorithm terminates. This corresponds to the GAMS/MINOS parameter "Row tolerance".
(default = 1e-6)

dens1 (real) LUSOL: density to start searching maxcol columns

The density at which the Markowitz strategy should search maxcol columns and no rows.
(default = 0.3)

dens2 (real) LUSOL: density to start searching 1 column

The density at which the Markowitz strategy should search only 1 column or (preferably) use a dense LU for all the remaining rows and columns.
(default = 0.6)

dmpfac (real) damping factor in backtracking linesearch


(default = 0.5)

elmax1 (real) LUSOL: max multiplier allowed in L during factor


(default = 10)

elmax2 (real) LUSOL: max multiplier allowed in L during update


(default = 10)

factim (real) basis reinversion time

Indicates the maximum number of CPU seconds between recalculation of the basis factors.
(default = 0.5)

invfrq (integer) basis reinversion frequency

Determines the maximum number of Lemke iterations between recalculation of the basis factors. This corresponds to the GAMS/MINOS parameter "Factorization frequency".
(default = 200)

invlog (integer) toggle Lemke inversion logging

A switch which requests LUSOL to generate a report with basis statistics following each refactorization.
(default = 1)

itch (integer) iteration refinement frequency

Indicates the frequency with which the factorization is checked. The number refers to the number of basis replacement operations between refinements. This corresponds to the GAMS/MINOS parameter "Check frequency".
(default = 25)

iterlim (integer) minor (Lemke) iterations limit
   Synonym: itrlim


(default = GAMS IterLim)

itlimt (integer) major (Newton) iterations limit

An upper bound on the number of Newton iterations. This corresponds to the GAMS/MINOS parameter "Major iterations".
(default = 100)

lcpdmp (integer) LCP dump (post-scaling)

A switch to generate a printout of the LCP data after scaling.
(default = 0)

lcpech (integer) LCP echo print (pre-scaling)

A switch to generate a printout of the LCP data before scaling, as evaluated.
(default = 0)

levout (integer) set output level

Sets the level of debug output written to the log and status files. The lowest meaningful value is -1 and the highest is 3. This corresponds, roughly, to the GAMS/MINOS parameter "Print level".
(default = 1)

lprint (integer) LUSOL: print level

The print level, < 0 suppresses output.
(default = 0)
<0suppresses output
0gives error messages
1gives debug output from some of the routines in LUSOL
>=2gives the pivot row and column and the no. of rows and columns involved at each elimination step in lu1fac

lusize (integer) LUSOL: multiplier for estimating memory requirements

Used to estimate the number of LU nonzeros which will be stored, as a multiple of the number of nonzeros in the Jacobian matrix.
(default = 5)

maxcol (integer) LUSOL: max cols to search for pivot element

In lu1fac is the maximum number of columns searched allowed in a Markowitztype search for the next pivot element. For some of the factorization, the number of rows searched is maxrow = maxcol - 1.
(default = 5)

maxeul (integer) Euler iteration limit


(default = 0)

mineul (integer) Euler iteration limit


(default = 0)

minstp (real) minimum step length


(default = 0.01)

norm (integer) norm to use in measuring deviation

Defines the vector norm to be used for evaluating epsilon(z). Acceptable values are 1, 2 or 3 which correspond to p = 1, 2 and +INF.
(default = 1)

nrsmax (integer) restart limit

Sets an upper bound on the number of restarts which the linear subproblem solver will undertake before giving up.
(default = 1)

pivlog (integer) toggle Lemke pivot logging

A switch to generate a status file listing of the Lemke pivot sequence.
(default = 0)

plinfy (real) infinity used by the solver

The value assigned for "plus infinity" ("+INF" in GAMS notation).
(default = 1e20)

scale (integer) turn on scaling at every iteration

Invokes row and column scaling of the LCP tableau in every iteration. This corresponds, roughly, to the GAMS/MINOS switch "scale all variables".
(default = 1)

small (real) LUSOL: absolute zero tolerance

The absolute tolerance for treating reals as zero.
(default = 3.0d-13)

spdadj (real) Euler adjustment factor


(default = 0.01)

uspace (real) LUSOL: factor limiting waste space in U

In lu1fac, the row or column lists are compressed if their length exceeds uspace times the length of either file after the last compression.
(default = 3)

utol1 (real) LUSOL: absolute tolerance U diagonal

The absolute tol for flagging small diagonals of U.
(default = 3.7d-11)

utol2 (real) LUSOL: relative tolerance U diagonal

The relative tol for flagging small diagonals of U.
(default = 3.7d-11)

ztolda (real) zero tolerance on coefficients

ztolpv (real) absolute zero tolerance on pivots

The absolute pivot tolerance. This corresponds, roughly, to the GAMS/MINOS parameter "Pivot tolerance" as it applies for nonlinear problems.

ztolrp (real) relative zero tolerance on pivots

The relative pivot tolerance. This corresponds, roughly, to the GAMS/MINOS parameter "Pivot tolerance" as it applies for nonlinear problems.

ztolz0 (real) absolute tolerance for installing cover elements


(default = 1e-6)

ztolze (real) feasibility tolerance

Used in the subproblem solution to determine when any variable has exceeded an upper or lower bound. This corresponds to GAMS/MINOS parameter "Feasibility tolerance".
(default = 1e-6)
