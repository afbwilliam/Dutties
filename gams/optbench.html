Detailed Descriptions of BENCH Options

allsolvers (integer) run all valid solvers

Indicator if all valid solvers for given modeltype should be run.
(default = 0)

cumulative (integer) apply limits cumulatively across runs of multiple solvers

Indicator if resource time and iteration limits are interpreted as total limits for all solvers or for individual solvers. For example if enabled and reslim=1500 then all solvers have a total of 1500 seconds. If not enabled, then each solver has a limit of 1500 seconds.
(default = 0)

dualcstol (real) dual complementary slackness tolerance: passed to EXAMINER

By dual CS we refer to complementary slackness between dual variables and the primal constraints.
(default = 1e-7)

dualfeastol (real) dual feasibility tolerance: passed to EXAMINER

This tolerance is used for the checks on the dual variables and the dual constraints.
(default = 1e-6)

examiner (integer) call EXAMINER to verify solution

Flag to call GAMS/EXAMINER to independently verify solution for feasibility and optimality.
(default = 0)

outlev (integer) log output level


(default = 2)
1BENCH summary log output only
2BENCH summary log output and log output of each solver

paver (integer) write PAVER trace files

Indicator if PAVER trace files should be written. Enabling causes a trace file solver.pvr to be written for each solver called. If the solver uses an option file, then the resulting file is solver-optnum.pvr, where optnum is the option file number. The files created can be submitted to the PAVER Server for automated performance analysis.
(default = 0)

paverex (integer) write PAVER trace files for the EXAMINER run

Indicator if PAVER trace files should be written for the Examiner run. Enabling causes a trace file solver-ex.pvr to be written for each solver called. If any Examiner check fails (independent or solver), then we return a model status 14 (no solution returned) and a solver status of 4 (terminated by solver). If no Examiner check is done, for example, because the return status is infeasible, then the status codes are returned as is. If the solver uses an option file, then the resulting file is solver-optnum-ex.pvr, where optnum is the option file number. The files created can be submitted to the PAVER Server for automated performance analysis.
(default = 0)

primalcstol (real) primal complementary slackness tolerance: passed to EXAMINER

By primal CS we refer to complementary slackness between primal variables and the dual constraints.
(default = 1e-7)

primalfeastol (real) primal feasibility tolerance: passed to EXAMINER

This tolerance is used for the checks on the primal variables and the primal constraints.
(default = 1e-6)

returnlastsol (integer) return solution from the last solver


(default = 0)

solvers (string) list of solvers to benchmark

solver[.n] Solver is the name of the GAMS solver that should be used, and n is the integer corresponding to optfile for the root node. If .n is missing, the optfile treated as zero (i.e., the solver) will not look for an options file. This is a required option.
